# Kanban App (Offline-First Portable App)

React + Tauri + TypeScript で構築された、オフラインファースト・完全ポータブル構成のカンバン型タスク管理デスクトップアプリケーションです。

## 🎯 概要・特徴
- **完全オフライン＆ポータブル動作**: レジストリを汚さず、すべてのデータや設定は `KanbanApp.exe`（実行ファイル）を置いたフォルダと同じ場所に自動保存されます。
- **インストール不要**: USBメモリや任意のフォルダ（デスクトップ等）にexeファイルをポンと置くだけで起動・運用が可能です。
- **ダークテーマ（Obsidian風）デザイン**: 視認性が高く、目に優しいUIを採用しています。

## 🚀 使い方ガイド

### 基本的な使い方
*   **タスクの追加**: 画面右下の `＋` ボタン、または `Ctrl + N` で新規タスクを作成できます。
*   **カンバン表示**: ドラッグ＆ドロップでタスクのステータス（未着手・進行中・完了）を変更できます。カラム内での並び替えも可能です。設定画面でカードのプレビュー表示（メモ・カテゴリ・作成日）のオン/オフを切り替えることで、カードサイズを調整できます。
*   **リスト表示**: カラムのヘッダーをクリックすると並び替えができ、「フィルタ」からカテゴリやステータスでの絞り込みができます。ステータスフィルタはチェックボックス形式で、複数のステータスを同時に選択して表示できます。
*   **タイムライン表示**: タスクに設定した着手予定日・期限をガントチャート形式で一覧表示します。
*   **画面の切り替え**: `Ctrl + T` を押すたびに、カンバン → リスト → タイムライン の3画面が巡回します。

### Today（今日やる）機能
*   タスクの作成・編集時に「今日やるタスク」にチェックを入れると、スター（⭐）マークがつきます。
*   ヘッダーの「今日」ボタンを押すと、Today設定されたタスクのみが表示されます。

### 検索とカスタムタブ
*   **キーワード検索**: ヘッダー右上の検索バー、または `Ctrl + F` でタスクのタイトル・メモから検索できます。
*   **カスタムタブ**: 設定画面から「特定のカテゴリ」だけを表示するタブを作成できます。作成したタブはヘッダーに並び、ドラッグ＆ドロップで順番を入れ替えられます。

### マウス・キーボード操作
*   **左クリック**: タスクを左クリックすると「詳細情報（閲覧専用モード）」が開きます。メモの内容は全量が表示され、画面全体をスクロールして閲覧できます。
*   **右クリック**: タスクを右クリックすると「編集モード」が直接開きます。
*   **保存ショートカット**: 編集モーダルを開いている間は `Shift + Enter` で変更を保存できます。

### 着手予定日・期限
*   タスクの作成・編集時に「着手予定日」と「終了予定日（期限）」を設定できます。
*   **どちらか一方を入力した場合、もう一方も必須**となります（両方入力 or 両方空）。
*   期限を過ぎたタスクには ⚠ マークが表示され、赤色で強調表示されます。

### タイムライン表示
タスクのスケジュールをガントチャート形式で確認・操作できます。
*   **未予定パネル**: 左端のアイコンボタンで未予定タスク一覧の表示/非表示を切り替えられます。パネルの開閉状態はアプリ再起動後も保持されます。
*   **タイムラインへの配置**: 未予定タスクをドラッグしてタイムライン上にドロップすると、着手予定日・期限が自動設定されます。
*   **バーの移動・伸縮**: タイムライン上のバーをドラッグで移動できます。バーの左端・右端をドラッグすると期間を伸縮できます。
*   **ソート**: ツールバーの「日付↑↓」ボタンで着手予定日の昇順/降順を切り替えられます。
*   **ズーム**: `+` / `-` ボタンで表示倍率（日 → 週 → 月）を3段階で切り替えられます。
*   **今日ボタン**: 「今日」ボタンを押すと、今日の日付の位置にスクロールします。
*   **重要度表示**: バーの左端に優先度に応じた色付きボーダーが表示されます（赤=高、黄=中、緑=低）。
*   **期限超過**: 期限を過ぎた未完了タスクは ⚠ マーク＋赤テキスト＋赤いリングで強調表示されます。
*   **完了タスクの自動非表示**: ステータスが「完了」で、最終更新日と期限の両方が当日より前のタスクは、タイムラインから自動的に非表示となります（ノイズ軽減のため）。
*   **未予定タスクの表示条件**: 設定画面で「過去Nヶ月以内に更新されたタスク」の範囲を変更できます（完了タスクは非表示）。

## 🛠️ 開発・ビルド方法

ソースコードからアプリをビルドしたり、開発モードで起動したりする際の手順です。

### 1. 依存関係のインストール
プロジェクトのルートディレクトリで以下のコマンドを実行します。
```bash
npm install
```

### 2. 開発モードでの起動
デスクトップアプリとして起動し、コードの変更を即座に反映（ホットリロード）させながら開発できます。
```bash
npm run tauri dev
```

### 3. 本番用ビルド（exeファイルの作成）
配布用の最適化された実行ファイルを作成します。
```bash
npm run tauri build
```
ビルドが完了すると、`src-tauri/target/release/` 配下に `kanbanapp.exe` が生成されます。

## 📦 アプリの配布（他者への展開）方法

このアプリは「完全ポータブル構成」をメインとして設計されています。インストーラを用いたインストール作業は推奨せず、**実行ファイル（exe）を直接配布する方法**が最も簡単で安全です。

### 1. 配布するファイル
開発側でビルドを行うと、以下のパスに実行ファイルが生成されます。
`src-tauri/target/release/app.exe`

この `app.exe` のファイル名を `KanbanApp.exe` などにリネームし、使ってほしい人に渡してください（ファイル単体で渡してOKです）。

### 2. 受け取ったユーザーが行うこと（配置例）
受け取ったユーザーは、好きな場所に新しいフォルダを作り、そこにexeファイルを置くだけです。

**【配置の推奨例】**
「ドキュメント」の中に `Kanban` というフォルダを作り、そこに置くイメージです。
```text
C:\Users\【ユーザー名】\Documents\
  └── Kanban\
       └── KanbanApp.exe    <-- 渡されたexeファイルをここに置く
```

### 3. 初回起動と自動生成されるデータ構造
配置した `KanbanApp.exe` をダブルクリックして起動すると、**exeと同じフォルダ内に自動的にデータファイルが作成されます。**

起動後のフォルダの中身は以下のようになります。
```text
C:\Users\【ユーザー名】\Documents\Kanban\
 ├── KanbanApp.exe      <-- アプリ本体
 ├── config.json        <-- （自動生成）アプリの設定やタブのデータ
 └── data/
      └── tasks.db      <-- （自動生成）すべてのタスクが保存されるSQLiteデータベース
```
※データはこのフォルダの中に完全に閉じ込められているため、PCのシステム領域（レジストリ等）を汚しません。USBメモリの中にフォルダごと入れて持ち運ぶことも可能です。

## 🗑️ アプリの削除（アンインストール）とバックアップ

すべてのデータがひとつのフォルダにまとまっているため、管理は非常にシンプルです。

### アプリを削除したい場合
インストーラを使っていないため、Windowsの「プログラムの追加と削除」からのアンインストールは不要です。
**`KanbanApp.exe` が入っているフォルダ（上記の例なら `Kanban/` フォルダ）を丸ごとゴミ箱に捨てるだけ**で、アプリ本体もタスクのデータも跡形もなく完全に消去されます。

### データのバックアップ・移行をしたい場合
パソコンを買い替える場合などは、`KanbanApp.exe` と同じフォルダにある **`config.json`** と **`data/` フォルダ** を一緒にコピーして、新しいパソコンに持っていくだけで、これまでの全データと設定をそのまま引き継ぐことができます。

## ⚠️ オフライン環境での注意事項（WebView2について）

このアプリはインターネットに接続されていない完全なオフライン環境でも動作します。
ただし、アプリの画面表示にWindows標準機能である「**Microsoft Edge WebView2 ランタイム**」を利用しています。

*   **Windows 11** または **最新までアップデートされたWindows 10** では、デフォルトで組み込まれているため問題なく動作します。
*   **極度に制限された閉域環境や古いWindows 10** の場合、WebView2が入っておらず起動時にエラーが出る場合があります。
    *   その際は、[Microsoft公式ページ](https://developer.microsoft.com/ja-jp/microsoft-edge/webview2/) より「エバーグリーン スタンドアロン インストーラー」を別のオンライン端末でダウンロードし、オフライン端末に事前インストールしてください。

## 🚀 将来のパフォーマンス改善案（技術的負債への備え）

本アプリはローカルのSQLiteデータベースを使用しており、数年にわたって毎日タスクを追加し続けても、数MB〜数十MB程度のデータ量に収まるため著しいパフォーマンス低下は起きにくい設計になっています。（画像等のバイナリデータはファイルシステムに分離して保存されるため、DB本体は軽量に保たれます。）

また、カンバンやタイムライン表示においては「完了タスクをNヶ月分のみ表示する」「最大表示件数を制限する」機能により、メモリの大量消費を防いでいます。

しかし、長期間（数年以上）の激しい運用によって万が一動作の遅延やファイルの肥大化が気になった場合は、以下の対処を将来の開発課題として想定しています。

### 1. データベースファイルの最適化（VACUUM）
*   **課題**: データの更新・削除を長期間繰り返すと、SQLiteファイル（`tasks.db`）内に空き領域（断片化）が生じ、ファイルサイズが無駄に大きくなることがあります。
*   **改善案**: アプリの起動時、または設定画面の操作によって、SQLiteの `VACUUM` コマンドを実行する機能を実装し、DBファイルを再構築して最適なサイズに圧縮します。

### 2. 古い完了タスクのパージ（アーカイブ化）
*   **課題**: 過去数年分の完了タスクは画面には表示されなくなりますが、DB内には存在し続けるため、全体のクエリ速度にわずかな影響を与える可能性があります。
*   **改善案**: 設定画面に「○年以上前の完了タスクを完全に削除する」または「別のバックアップデータとして退避（アーカイブ）する」機能を追加し、メインのタスクデータベースを常に軽量な状態に保ちます。
